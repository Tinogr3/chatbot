# Cómo arrancar la aplicación (backend + frontend)

## Opción rápida: un solo comando

Desde la raíz del proyecto:

  ./run.sh

Levanta Redis (si aplica), el backend FastAPI, el worker Celery y el frontend Next.js. Al salir (Ctrl+C) se detienen los servicios gestionados por el script.

Puertos por defecto: backend 8000, frontend Next.js 3000. Ejemplo de override:

  BACKEND_PORT=9000 FRONTEND_PORT=3001 ./run.sh

## Opción con dos terminales

### 1. Entorno Python e instalación (una sola vez)

  python3 -m venv venv
  source venv/bin/activate
  pip install -r backend/requirements.txt

### 2. Arrancar el backend (terminal 1)

Desde la raíz del proyecto:

  source venv/bin/activate
  export PYTHONPATH="$(pwd):${PYTHONPATH:-}"
  uvicorn backend.main:app --reload --host 0.0.0.0 --port 8000

Comprobar: http://localhost:8000/health → respuesta JSON con estado ok.

Nota: si usas subida de archivos o vídeo con cola asíncrona, necesitas Redis y un worker Celery (como hace run.sh). Para solo probar chat y endpoints síncronos puede bastar el backend solo.

### 3. Arrancar el frontend (terminal 2)

  cd frontend
  npm install
  npm run dev

Abrir en el navegador: http://localhost:3000 (puerto por defecto de `next dev`).

Si el backend no está en http://localhost:8000, definir antes de `npm run dev`:

  export NEXT_PUBLIC_BACKEND_URL=http://HOST:PUERTO

## Opcional

- NEXT_PUBLIC_BACKEND_URL: URL pública del API vista por el navegador (Next.js inyecta las variables con prefijo NEXT_PUBLIC_ en el cliente).
- Variables del backend: archivo .env en la raíz del repo o variables como GOOGLE_API_KEY, GOOGLE_APPLICATION_CREDENTIALS, BUCKET_NAME (ver backend/config.py).

## Ngrok

Para exponer el frontend: por ejemplo `ngrok http 3000` (o el puerto donde corre `npm run dev`).

Para exponer solo el API: `ngrok http 8000`.
